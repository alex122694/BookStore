# 🚀 前端部署與系統啟動教學

本文件將會為您詳細說明專案的前後端啟動方式，主要分為「**開發環境（前後端分離獨立啟動）**」與「**生產環境（將前端打包好的靜態檔案整合至 Spring Boot 後端）**」兩種情境模式。

---

## 模式一：開發環境 (前後端分離啟動)

在開發階段，我們通常將前端與後端分別運行在不同的伺服器中，以便享有前端的「熱更新 (Hot-Reload)」以及後端的快速除錯。

### 1. 啟動後端 (Spring Boot)
1. 開啟終端機 (Terminal / CMD)，進入 `bookstore-backend/` 目錄。
2. 透過 Maven 啟動：
   ```bash
   mvnw spring-boot:run
   ```
3. 後端伺服器預設會於 `http://localhost:8080` 上運行。

### 2. 啟動前端 (Vue + Vite)
1. 開啟一個**新的終端機視窗**，並進入 `bookstore-frontend/` 目錄。
2. 進行前端套件模組安裝，並啟動開發測試伺服器：
   ```bash
   npm install
   npm run dev
   ```
3. 成功啟動後，前端開發伺服器預設會運行在 `http://localhost:5173`。

> **💡 注意**：開發環境中，前端透過 `vite.config.js` 內的 Proxy 自動代理機制，將發送至 `/api` 的請求完美轉交給 `http://localhost:8080`，避免了跨域請求 (CORS) 問題。

## 模式二：整合發布環境 (將前端打包放入後端)

當系統進入實際上線部署、或是希望讓客戶不需開啟兩台伺服器也能使用時，可以將 Vue 前端專案編譯為純靜態網頁檔案 (HTML / CSS / JS)，並將這些檔案內置於 Spring Boot 後端當中。
只要啟動後端，即可同時提供前後端網站服務，兩者完美共用同一個連接埠 (Port 8080)。

### 具體步驟說明

#### 第一步：編譯前端靜態檔案 (Build)
請開啟終端機，並進入前端專案目錄進行打包：
```bash
cd bookstore-frontend
npm run build
```
指令執行完畢後，Vite 框架會在 `bookstore-frontend/` 根目錄下自動產生一個名為 `dist` 的資料夾。
這包資料夾便包含了被 Vite 高度壓縮與優化的專案靜態檔案。

#### 第二步：移轉靜態檔案至後端專案中
請找到 Spring Boot 負責提供靜態資源的專屬資料夾路徑，通常位於：
👉 `bookstore-backend/src/main/resources/static/`

請將剛才編譯出來的 `bookstore-frontend/dist/` 資料夾內的所有內容（包含 `index.html` 及 `assets/` 目錄等），**全部複製並貼上**到後端的 `static/` 這個資料夾中。

> ⚠️ 若您發現 `src/main/resources/static/` 目錄不存在，請自行手動新建該資料夾。

#### 第三步：重新啟動後端專案
回到後端專案的根目錄中，如同平常一樣重新編譯並啟動您的後端伺服器：
```bash
cd bookstore-backend
mvnw clean spring-boot:run
```

#### 第四步：檢視成果
待後端伺服器啟動完成後，開啟瀏覽器並直接造訪後端伺服器位址：
👉 **`http://localhost:8080`**

這時您會發現，畫面上呈現的就是原先在 5173 Port 執行的 Vue 前端應用！所有 API 通訊與網頁導覽皆完全由 Spring Boot 一手包辦送出，代表整合部署已經大功告成！
